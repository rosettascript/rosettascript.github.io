## Implementation Notes

### Design:
- **Accordion-style expandable Q&A** (better UX, saves space)
- **Clear category grouping** with visual separation and collapsible sections per category
- **Category icons/colored tags** for faster scanning (e.g., 🔒 Privacy, 🛠️ Tools, 📱 Usage)
- **Mobile-responsive design** with touch-friendly accordion controls
- **Search/Filter functionality**: Add a search input to filter questions by keyword (improves UX and keeps users on-page longer, which is good for SEO metrics like time on page and engagement)

### Accessibility (WCAG 2.1 AA):
- **ARIA attributes**: Use `aria-expanded`, `aria-controls`, `aria-labelledby` for accordion items
- **Keyboard navigation**: Support Tab, Enter, and Space keys for expanding/collapsing
- **Screen reader support**: Proper heading hierarchy (h2 for categories, h3 for questions)
- **Focus management**: Visible focus indicators for keyboard users
- **Semantic HTML**: Use `<button>` for accordion triggers, not `<div>` or `<span>`

### Performance:
- **Lazy-load icons**: Load category icons and tool icons on-demand to avoid impacting initial page load
- **Code splitting**: Consider lazy-loading FAQ component if it's large
- **Image optimization**: Use WebP format for icons with fallbacks
- **Minimize re-renders**: Use React.memo for FAQ items if needed
- **JSON-LD script loading**: Consider lazy-loading the FAQ JSON-LD `<script>` tag at the bottom of `<body>` if the page has many other heavy scripts
- **Smooth animations**: Add smooth CSS transitions for expanding/collapsing accordion items to enhance perceived performance (especially on mobile)

### Schema Rich Snippet Enhancement:
- **Image fields**: Add `image` property for tool-specific FAQs (e.g., Word to HTML converter icon)
- **Entity ordering**: Ensure `mainEntity` array order matches visual order on page
- **Additional properties**: Consider adding `datePublished` and `dateModified` if FAQ content changes over time
- **BreadcrumbList**: Link FAQ to site breadcrumb structure if applicable
- **Visual hierarchy**: Category headers, question styling, and answer formatting for easy scanning

### Link Formatting Consistency:
- **Standard format**: [Tool Name](/path/) - Use markdown links consistently
- All tool mentions should link to their respective pages
- Blog post links use descriptive anchor text with keywords
- **Internal links**: Use relative paths (React Router will handle them) - ensure all relative links resolve correctly on production
- **Blog posts**: Consider canonical URLs or absolute links to avoid duplicate content issues
- Absolute URLs only in JSON-LD schema for Google
- **Answer format**: Start with "Use our [tool name]" for active voice and consistency

### JSON-LD Schema Format Standardization:
- **Standard format**: "Use our [tool name] at [URL] to [action]..."
- Tool name + link mentioned first in each answer
- Consistent structure across all tool-related questions
- Optional "url" field added for questions that link to specific tools (enables deep linking in Google search)

### Placement:

- Should be visible and crawlable (not hidden behind excessive JS)

### Internal Linking Strategy:
- Link to tool pages when tools are mentioned
- Link to blog posts for advanced topics (RUC, web scraping, Word to HTML)
